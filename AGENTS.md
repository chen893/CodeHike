通过 CodeHike 最新文档 https://codehike.org/docs，在当前项目下生成教程时，默认按 `Build your own react` 的交互范式实现。

硬规则：

1. 布局优先参考 `Build your own react`
左侧是固定在视口内的代码舞台，右侧是独立滚动的文章区。
不要让整页滚动；桌面端应优先让文章栏单独 `overflow-y: auto`。

2. 代码区必须是“同一个面板持续变化”
不要用切换整块代码组件的方式实现步骤切换。
应保持同一个代码面板实例，只更新内部 code 数据，这样 CodeHike 的增删与 token 过渡动画才能生效。

3. 教程节奏要细
不要把一大段解释配一大块代码改动。
应把文章拆成更小的步骤，让每次滚动只引入一处或少量代码变化，接近逐句讲解。

4. 代码变化要可感知
参考 `Build your own react` 时，代码变化不只是内容替换。
新增行、删除行、focus 区域都应该有明显但克制的动画或视觉反馈。

5. 避免双滚动冲突
代码面板默认不要再有独立纵向滚动，也不要在步骤切换时主动对代码区执行内部 `scrollTo`，除非明确验证不会与文章滚动冲突。

6. 小样验证时，先验证“交互结构”，再扩正式教程
先做一个很小的单页原型，确认布局、滚动、代码变化方式符合预期，再开始完整教程内容。

7. UI 气质参考
整体更像编辑式教程，而不是产品营销页：
左侧深色、安静、克制；
右侧暖白、长文阅读排版；
减少卡片感、玻璃感、强边框和过度装饰。

如果用户说“参考 Build your own react”，默认按以上规则执行，除非用户明确要求其他结构。
